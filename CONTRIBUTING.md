# Contributing to UniZ

First off, thank you for considering contributing to UniZ. It's people like you that make UniZ such a great tool.

## Code of Conduct

This project and everyone participating in it is governed by the UniZ Code of Conduct. By participating, you are expected to uphold this code.

## Workflow

1. Fork the repository.
2. Clone the project to your own machine.
3. Create a branch locally with a succinct but descriptive name.
4. Commit changes to the branch.
5. Push changes to your fork.
6. Open a Pull Request in our repository.

## Elite Standards

- Clean Code: Write code that is easy to read, simple to understand, and maintainable.
- Testing: Add tests for your changes. We aim for high coverage.
- Commits: Use semantic commit messages (e.g., feat: add user login, fix: resolve overflow issue).
- Documentation: Update documentation if you change behavior or add features.

## Bug Reports

- Search for existing issues.
- Template: Use the Bug Report template to provide all necessary details.

## Feature Requests

- Discussion: Open an issue to discuss the feature before starting work.
- Value: Explain how this feature adds value to the UniZ ecosystem.
