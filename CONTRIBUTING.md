# CONTRIBUTING

Thank you for your interest in contributing to UniZ. We welcome contributions from the community and appreciate your help in making this project better.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on GitHub. Include as much detail as possible, such as steps to reproduce the issue, the expected behavior, and the actual behavior.

### Requesting Features

If you have an idea for a new feature, please open an issue to discuss it. We value community feedback and want to know how we can improve UniZ.

### Submitting Pull Requests

We welcome pull requests for bug fixes, new features, and improvements to the documentation. Before submitting a pull request, please ensure that:

1. Your code follows the project's coding standards.
2. You have added tests for your changes.
3. Your changes do not break any existing functionality.

## Development Process

1. Fork the repository.
2. Create a new branch for your changes.
3. Make your changes and commit them with a descriptive message.
4. Push your changes to your fork.
5. Open a pull request against the main repository.

## Community

Join our community to discuss the project, ask questions, and share your ideas. We value inclusive and respectful communication and expect all members to follow our Code of Conduct.
