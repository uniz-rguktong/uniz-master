---
title: "Subjects"
description: "Manage subjects and semesters, and handle student subject registration."
---

All subjects and semester endpoints require authentication. Administrative operations (add, update, delete) require elevated roles as noted per endpoint.

---

## List subjects

`GET /academics/subjects`

Returns all subjects configured in the system.

**Auth required:** Yes

### Example

```bash
curl --request GET \
  --url https://api.uniz.rguktong.in/api/v1/academics/subjects \
  --header 'Authorization: Bearer <token>'
```

**200 OK**

```json
{
  "success": true,
  "subjects": [
    {
      "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
      "code": "E1-SEM-1-CSE-1",
      "name": "Engineering Mathematics I",
      "credits": 4,
      "branch": "CSE",
      "year": "E1",
      "semesterId": "SEM-1"
    }
  ]
}
```

### Response fields

<ResponseField name="subjects" type="object[]" required>
  <Expandable title="subject properties">
    <ResponseField name="id" type="string">
      UUID of the subject.
    </ResponseField>
    <ResponseField name="code" type="string">
      Subject code (e.g. `E1-SEM-1-CSE-1`).
    </ResponseField>
    <ResponseField name="name" type="string">
      Full subject name.
    </ResponseField>
    <ResponseField name="credits" type="number">
      Credit hours.
    </ResponseField>
    <ResponseField name="branch" type="string">
      Branch this subject belongs to (e.g. `CSE`).
    </ResponseField>
    <ResponseField name="year" type="string">
      Academic year this subject is offered (e.g. `E1`).
    </ResponseField>
    <ResponseField name="semesterId" type="string">
      Semester identifier (e.g. `SEM-1`).
    </ResponseField>
  </Expandable>
</ResponseField>

## Add a subject

`POST /academics/subjects/add`

Creates a new subject.

**Auth required:** Yes (dean, director, webmaster)

### Request body

<ParamField body="code" type="string" required>
  Unique subject code (e.g. `E1-SEM-1-CSE-1`).
</ParamField>

<ParamField body="name" type="string" required>
  Full subject name (e.g. `Engineering Mathematics I`).
</ParamField>

<ParamField body="credits" type="number" required>
  Number of credit hours.
</ParamField>

<ParamField body="branch" type="string" required>
  Branch the subject belongs to (e.g. `CSE`).
</ParamField>

<ParamField body="year" type="string" required>
  Academic year the subject is offered (e.g. `E1`).
</ParamField>

<ParamField body="semesterId" type="string" required>
  Semester the subject is part of (e.g. `SEM-1`).
</ParamField>

### Example

```bash
curl --request POST \
  --url https://api.uniz.rguktong.in/api/v1/academics/subjects/add \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "code": "E1-SEM-1-CSE-1",
    "name": "Engineering Mathematics I",
    "credits": 4,
    "branch": "CSE",
    "year": "E1",
    "semesterId": "SEM-1"
  }'
```

**200 OK**

```json
{
  "success": true,
  "message": "Subject added successfully",
  "subject": {
    "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "code": "E1-SEM-1-CSE-1",
    "name": "Engineering Mathematics I",
    "credits": 4,
    "branch": "CSE",
    "year": "E1",
    "semesterId": "SEM-1"
  }
}
```

## Update a subject

`PUT /academics/subjects/:id`

Updates an existing subject by its UUID.

**Auth required:** Yes (dean, director, webmaster)

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the subject to update.
</ParamField>

### Request body

All fields are optional. Provide only the fields you want to change.

<ParamField body="name" type="string">
  Updated subject name.
</ParamField>

<ParamField body="credits" type="number">
  Updated credit hours.
</ParamField>

<ParamField body="code" type="string">
  Updated subject code.
</ParamField>

### Example

```bash
curl --request PUT \
  --url https://api.uniz.rguktong.in/api/v1/academics/subjects/3fa85f64-5717-4562-b3fc-2c963f66afa6 \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "credits": 5
  }'
```

**200 OK**

```json
{
  "success": true,
  "message": "Subject updated successfully"
}
```

## Initialize a semester

`POST /academics/semester/init`

Creates and activates a new semester.

**Auth required:** Yes (director, dean)

### Request body

<ParamField body="semesterId" type="string" required>
  Identifier for the new semester (e.g. `SEM-2`).
</ParamField>

<ParamField body="startDate" type="string" required>
  ISO 8601 start date (e.g. `2026-07-01`).
</ParamField>

<ParamField body="endDate" type="string" required>
  ISO 8601 end date (e.g. `2026-12-31`).
</ParamField>

### Example

```bash
curl --request POST \
  --url https://api.uniz.rguktong.in/api/v1/academics/semester/init \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "semesterId": "SEM-2",
    "startDate": "2026-07-01",
    "endDate": "2026-12-31"
  }'
```

**200 OK**

```json
{
  "success": true,
  "message": "Semester initialized",
  "semester": {
    "id": "sem-uuid-002",
    "semesterId": "SEM-2",
    "status": "active",
    "startDate": "2026-07-01T00:00:00.000Z",
    "endDate": "2026-12-31T00:00:00.000Z"
  }
}
```

## Register for subjects

`POST /academics/student/register`

Registers the authenticated student for one or more subjects in the current semester.

**Auth required:** Yes (student)

### Request body

<ParamField body="subjectIds" type="string[]" required>
  Array of subject UUIDs to register for.
</ParamField>

### Example

```bash
curl --request POST \
  --url https://api.uniz.rguktong.in/api/v1/academics/student/register \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "subjectIds": [
      "3fa85f64-5717-4562-b3fc-2c963f66afa6",
      "7cb96g75-6828-5673-c4gd-3d074g77bgb7"
    ]
  }'
```

**200 OK**

```json
{
  "success": true,
  "message": "Subjects registered successfully",
  "registered": 2
}
```

### Error responses

| Status | Meaning |
|--------|---------|
| `400 Bad Request` | One or more subject IDs are invalid or the registration window is closed. |
| `409 Conflict` | Student is already registered for one or more of the provided subjects. |

## Get current registered subjects

`GET /academics/student/current/:studentId`

Returns the subjects a student is currently registered for in the active semester.

**Auth required:** Yes

### Path parameters

<ParamField path="studentId" type="string" required>
  Student roll number (e.g. `O210008`). Students may only query their own record; admin roles may query any student.
</ParamField>

### Example

```bash
curl --request GET \
  --url https://api.uniz.rguktong.in/api/v1/academics/student/current/O210008 \
  --header 'Authorization: Bearer <token>'
```

**200 OK**

```json
{
  "success": true,
  "semesterId": "SEM-1",
  "subjects": [
    {
      "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
      "code": "E3-SEM-1-CSE-1",
      "name": "Operating Systems",
      "credits": 4
    }
  ]
}
```

### Response fields

<ResponseField name="semesterId" type="string">
  The active semester these registrations belong to.
</ResponseField>

<ResponseField name="subjects" type="object[]">
  List of currently registered subjects.
</ResponseField>
